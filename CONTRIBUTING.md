# Contributing to Nabi OpenCore

Thank you for helping build an open, stable, and trustworthy runtime.

License and IP
- Code and docs are Apache‑2.0 (see LICENSE). By contributing, you agree to license your contributions under Apache‑2.0.
- We use the Developer Certificate of Origin (DCO); no CLA is required.

Developer Certificate of Origin (DCO)
Sign your commits with:

```
Signed-off-by: Your Name <you@example.com>
```

You can add this automatically with `git commit -s`.
Learn more: https://developercertificate.org/

How to contribute
1. Discuss: Open an issue or draft an RFC for substantial changes.
2. Develop: Fork, create a feature branch, add tests and docs.
3. Test: Ensure unit/integration tests pass locally.
4. Submit: Open a PR with a clear description, motivation, and migration notes.
5. Review: Address feedback; maintainers merge with squash or conventional commits.

Quality bar
- Tests for new behavior; non‑trivial bug fixes include regression tests.
- Documentation updates with user‑facing changes.
- Security‑sensitive changes must include threat considerations.

Style and tooling
- Follow existing code style and project conventions.
- Keep changes minimal and focused; avoid drive‑by refactors.

Security disclosures
- Do not open public issues for vulnerabilities. See SECURITY.md for private reporting.

Community expectations
- Follow the Code of Conduct; be respectful and constructive.
